# BookManager / 藏书阁

一款面向长期个人使用的本地优先桌面书库，用于管理藏书、分类、阅读状态、章节进度、封面、电子书附件和阅读笔记。

## 功能亮点

- **本地优先：** 书目保存在 SQLite 数据库中，无需账号、云服务或环境变量。
- **分类管理：** 支持最多 5 层树状分类，并提供不可删除的“未分类”系统目录。
- **阅读管理：** 手动选择“想读 / 在读 / 已读”，根据章节页码自动计算已读页数和百分比。
- **完整书目信息：** 记录书名、副标题、原书名、作者、译者、ISBN、出版社、版次、系列、语言、载体、页数、简介、标签和评分。
- **章节与笔记：** 新章节自动承接上一章终止页，每章可单独备注，并提供整本书的阅读笔记。
- **封面与附件：** 从本地或 URL 导入封面，把电子书和相关文件复制进对应书籍文件夹长期保存。
- **安全删除：** 书籍先进入回收站，只有再次确认永久删除后才清理数据库记录和受管理文件。
- **迁移与备份：** 数据库、自动备份、JSON 导出、封面和附件统一存放在可迁移的书库根目录。
- **桌面体验：** 提供搜索、筛选、排序、网格/列表视图、阅读统计、深浅主题、右键菜单和低窗口高度适配。

## 运行要求

- Windows 10 或 Windows 11
- Node.js 20 或更高版本
- npm 10 或更高版本
- 首次安装依赖时需要网络；日常书库管理可完全离线使用

当前仓库提供源码运行方式，暂未提供安装包。

## 快速开始

克隆仓库并启动开发模式：

```powershell
git clone https://github.com/xwbperson/BookManager.git
Set-Location BookManager
npm install
npm run dev
```

`npm run dev` 会在 `127.0.0.1:5173` 启动 Vite，并自动打开 Electron 窗口。关闭 Electron 后，开发服务器也会退出。

运行生产构建：

```powershell
npm run build
npm start
```

应用不需要 `.env` 文件或其他环境变量。

## 常用操作

### 管理书籍

在主页添加书籍后，可以通过书籍卡片右键菜单快速编辑、移动分类或移入回收站；键盘用户可聚焦卡片后按 `Shift+F10` 打开相同菜单。

阅读状态始终由用户手动选择，章节进度不会擅自改变“想读 / 在读 / 已读”状态。总进度按照每章的起止页和当前页自动汇总。

### 管理封面和附件

- 本地封面会复制到书籍文件夹。
- URL 封面会先下载并保存到本地，后续展示不依赖原网址。
- 添加的电子书或文档会复制到本书的 `attachments/` 目录，原始外部文件不再作为长期依赖。
- 可以从书籍详情页打开附件或整本书的文件夹。

### 更换书库位置

在“设置”中选择其他目录时，可以：

- 把当前书库迁移到一个空目录；
- 打开一个已经包含 `book-manager.sqlite` 的现有书库。

新旧书库目录不能互相包含，避免迁移时递归复制文件。

## 书库根目录

首次运行默认使用当前用户“文档”目录下的 `BookManagerLibrary`。长期数据全部位于这个根目录，Electron 的应用配置目录只保存根目录位置。

```text
BookManagerLibrary/
├─ book-manager.sqlite
├─ backups/
├─ exports/
└─ books/
   ├─ 未分类/
   ├─ 分类/子分类/书名__书籍ID/
   │  ├─ book.json
   │  ├─ cover.jpg
   │  └─ attachments/
   └─ .trash/
```

修改书名、移动分类、移入回收站或恢复书籍时，受管理的书籍文件夹会同步移动。每本书的 `book.json` 是便于查看的元数据快照，SQLite 数据库仍是应用的主要数据源。

## 导入导出与备份

- **JSON 导出：** 从侧边栏点击“导出”，版本化 JSON 会写入根目录的 `exports/`。
- **JSON 内容：** 包含书目、分类、章节、章节备注、附件元数据、封面数据、主题和年度阅读目标。
- **附件限制：** JSON 不嵌入附件文件本体；需要完整离线备份时，请复制整个书库根目录。
- **导入保护：** 导入 JSON 前会创建数据库恢复备份，再覆盖当前书目数据。
- **自动备份：** 应用每天启动时自动备份一次数据库，危险操作前也会创建备份，并保留最近 14 份自动备份。

## 技术栈

- [Electron 33](https://www.electronjs.org/)：桌面窗口、文件选择、系统文件打开和进程隔离
- [React 18](https://react.dev/)：界面状态与组件
- [Vite 6](https://vite.dev/)：开发服务器与生产构建
- [sql.js](https://sql.js.org/)：SQLite 数据库存储
- [Lucide React](https://lucide.dev/)：界面图标

渲染进程只能通过 `preload.js` 暴露的受限 IPC 接口访问本地文件和数据库；数据库写入采用临时文件替换方式，降低异常退出导致文件损坏的风险。

## 项目结构

```text
BookManager/
├─ main.js                 # Electron 主进程与 IPC
├─ preload.js              # 渲染进程安全桥接
├─ renderer/
│  ├─ main.jsx             # React 界面与交互
│  └─ styles.css           # 主题和响应式布局
├─ src/
│  ├─ database.js          # SQLite、书籍文件和设置持久化
│  ├─ backup.js            # JSON 备份格式与输入校验
│  └─ library-root.js      # 书库根目录选择与迁移
├─ scripts/                # 开发启动和回归测试脚本
├─ docs/adr/               # 关键架构决策记录
├─ book-manager.html       # 早期单文件原型，仅作设计参考
└─ vite.config.js
```

## 开发与验证

```powershell
npm run check
npm run smoke
npm run build
npm run electron:smoke
npm run ui:regression
```

| 命令 | 用途 |
|------|------|
| `npm run check` | 检查主进程、预加载脚本、数据模块和测试脚本语法 |
| `npm run smoke` | 验证 SQLite、备份格式、附件、封面、书籍目录和根目录迁移 |
| `npm run build` | 生成 `dist/` 生产资源 |
| `npm run electron:smoke` | 启动生产模式 Electron 并验证主窗口 |
| `npm run ui:regression` | 在真实 Electron 窗口中验证滚动、菜单、章节、附件、导入导出、迁移和回收站生命周期 |

## 数据与安全说明

- 文本输入、确认和危险操作使用应用内对话框；系统原生窗口只用于选择文件或目录。
- 导入数据会校验目录深度、重复 ID、日期、文件名、章节数量和附件数量等边界。
- URL 封面仅接受图片响应，并限制下载与图片尺寸。
- 不要把真实书库根目录、SQLite 数据库、电子书附件或私人阅读笔记提交到 Git 仓库。
